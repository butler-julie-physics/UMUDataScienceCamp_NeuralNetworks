## Introduction

[Machine learning](https://en.wikipedia.org/wiki/Machine_learning) is a subset of artificial intelligence, which draws heavily from the fields of computer science and statistics. The goal of machine learning is to create an algorithm that can learn the patterns that exist in a data set such that the algorithm can recreate those patterns with new data. In this case we will be using machine learning to perform a task called [classification](https://en.wikipedia.org/wiki/Classification), where the goal is to sort data into the correct category. 

In this activity, the machine learning algorithm we will be using is called a [neural network](https://en.wikipedia.org/wiki/Neural_network_(machine_learning)). Neural networks are modeled after the body's nervous system, where connections of different strengths are made between units called [neurons](https://en.wikipedia.org/wiki/Artificial_neuron). In this case, the neurons are arranged in rows, with connections between neurons in different rows. When a neuron network learns from data, it strengths connections between some neurons and weakens the connections between others to make sure output of the neural network best matches to data it is learning from.

### Important Terms
* [Neuron](https://en.wikipedia.org/wiki/Artificial_neuron): An artificial neuron is a simple computing unit that takes in several numbers, multiplies each by a "weight" that signals how important it is, adds them all up (along with an extra adjustment called a bias), and then runs the result through a function to decide what output to send on. It's loosely inspired by how nervous system cells work, and stacking many of these neurons together is what lets a neural network learn to recognize patterns.
* Layer: A neural network layer is a row of artificial neurons working side by side, where each neuron receives the same set of inputs but processes them with its own weights to produce its own output. The layer's combined outputs then become the inputs for the next layer. By stacking layers one after another, a network can build up from simple patterns in early layers to more complex ones deeper in, which is how it learns to handle tasks like recognizing images or understanding text.
    * Input Layer: An input layer is the first layer of a neural network, where the raw data enters.An input layer is the first layer of a neural network, where the raw data enters. Its job is simply to receive the input values and pass them on to the next layer, without doing any calculations of its own. Each node in the input layer represents one feature (input) of the data.
    * [Hidden Layer](https://en.wikipedia.org/wiki/Hidden_layer): A hidden layer is any layer between the input and output layers, where the actual processing happens. Its neurons take the values from the previous layer, apply their weights, biases, and activation functions, and pass the results forward. They're called "hidden" because you don't directly see their outputs the way you see the network's input and final answer. A network can have one or many hidden layers, and having several is what makes a network "deep" and able to learn more complex patterns.
    * Output Layer: An output layer is the final layer of a neural network, where the answer comes out. It takes the values from the last hidden layer and turns them into the network's final result. The number of neurons depends on the task: a network sorting images into 10 categories would have 10 output neurons (one per category), while one predicting a single number might have just one. The output is often shaped so it's easy to interpret, such as probabilities that add up to 100%.
* Batch Size: Batch size is the number of training examples a neural network processes at once before updating its weights. Instead of learning from one example at a time (slow) or all examples at once (memory-heavy), the data is split into batches—say, 32 or 64 examples each. The network makes predictions for a whole batch, measures how wrong it was on average, and then adjusts its weights once based on that batch. Smaller batches update more often but more noisily; larger batches are smoother but need more memory.
* Epoch: An epoch is one complete pass through the entire training dataset. If you have 1,000 training examples and a batch size of 100, it takes 10 batches to get through all of them—that's one epoch. Training usually involves many epochs (sometimes dozens or hundreds), because the network needs to see the data repeatedly to gradually improve its weights. Too few epochs and the network hasn't learned enough; too many and it can start memorizing the training data instead of learning general patterns.
* [Activation Function](https://en.wikipedia.org/wiki/Activation_function): An activation function is the step inside a neuron that decides what output to send on after the inputs have been weighted and summed. Its key job is to add non-linearity, meaning it lets the network bend and curve to learn complex patterns rather than just straight-line relationships—without it, even a deep network could only do simple linear math. A common example is ReLU, which passes positive values through unchanged and turns negative values into zero. Others include the sigmoid, which squashes any number into a value between 0 and 1.
* [Learning Rate](https://en.wikipedia.org/wiki/Learning_rate): The learning rate is a setting that controls how big a step the network takes when adjusting its weights during training. After measuring how wrong it was, the network nudges its weights in the right direction, and the learning rate decides how large that nudge is. If it's too high, the network takes huge steps and may overshoot the best answer or never settle; if it's too low, learning is very slow and can get stuck. Finding a good learning rate—often a small number like 0.01—is one of the most important parts of training a neural network well.
* [Loss Function](https://en.wikipedia.org/wiki/Loss_function): A loss function is a formula that measures how wrong the network's predictions are compared to the correct answers. It produces a single number—the "loss"—where a high value means the predictions are far off and a low value means they're close. The whole point of training is to make this number as small as possible, since the network adjusts its weights specifically to reduce the loss. Different tasks use different loss functions: for example, one type is common for predicting numbers, while another is used for sorting things into categories.
* Loss: Loss is the single number that tells you how wrong the network's prediction is compared to the correct answer. It's the output of the loss function—a high loss means the prediction is far off, and a low loss means it's close. During training, the network's entire goal is to make the loss as small as possible by adjusting its weights. You can think of loss as a "wrongness score" that the network is constantly trying to lower.
* Accuracy: Accuracy is a simple measure of how often the network gets the right answer, usually written as a percentage. You calculate it by dividing the number of correct predictions by the total number of predictions—so if the network classifies 90 out of 100 images correctly, its accuracy is 90%. It's an easy way to judge how well a network is doing, especially for sorting things into categories. Unlike loss, which measures how wrong predictions are, accuracy just counts how many are right or wrong, with no partial credit.
* [Features](https://en.wikipedia.org/wiki/Feature_(machine_learning)): Features are the individual pieces of information you feed into a neural network as input—the measurable characteristics it uses to make a prediction. For example, to predict a house's price, the features might be its square footage, number of bedrooms, and age. In an image, each pixel's brightness is a feature. Choosing good features matters a lot, because the network can only learn from the information you give it, so features that capture what's truly important to the task help it make better predictions.
* Output: In a neural network, the output is the final answer the network produces after processing the input. It comes out of the output layer and takes whatever form the task needs: a category label (like "cat" or "dog"), a number (like a predicted price), or a set of probabilities showing how confident the network is in each possible answer. The output is then compared against the correct answer during training so the network can measure its error and improve.
* [Parameters](https://en.wikipedia.org/wiki/Parameter#Artificial_intelligence): Parameters are the values inside a neural network that it learns and adjusts during training—mainly the weights and biases. They're what actually get tuned as the network practices: every time it sees how wrong it was, it nudges these values to do better next time. A network can have anywhere from a few hundred to billions of parameters, and more parameters generally let it learn more complex patterns. In short, parameters are the network's "knowledge," storing everything it has learned from the data.
* [Hyperparameters](https://en.wikipedia.org/wiki/Hyperparameter_(machine_learning)): A hyperparameter is a setting you choose before training begins, rather than something the network learns on its own. Examples include the learning rate, the batch size, the number of epochs, and how many hidden layers the network has. These differ from parameters (the weights and biases), which the network adjusts automatically during training—hyperparameters are knobs you set by hand to control how the training works. Picking good hyperparameters is often a matter of experimenting and tuning, since the right choices can make the difference between a network that learns well and one that struggles.
* [Hyperparameter Tuning/Optimization](https://en.wikipedia.org/wiki/Hyperparameter_optimization): Hyperparameter tuning is the process of experimenting with different hyperparameter settings to find the combination that makes the network learn best. Since you can't know the perfect learning rate, batch size, or number of layers in advance, you try different values, train the network, and see which choices give the best results. This can be done by hand, by systematically testing many combinations (like a grid of options), or with automated search methods. It's often one of the more time-consuming parts of building a neural network, but good tuning can significantly improve how well the network performs.


## Coding a Neural Network in Python
### First Attempt at a Neural Network

First we will attempt a very basic neural network. The only feature we will use to predict a passenger's survival is the age of the passenger. We will then [split the data into training and test sets](https://builtin.com/data-science/train-test-split). The neural network uses the training set to learn the patterns in the data, to determine the best value of its parameters in order to most accurately recreate the training data. We will then use the test data to determine if the network is able to _generalize_ to data it has never seen before. A machine learning model that is not able to generalize to data it has never seen before is not useful. To split the data we will use the function [`train_test_split`](https://scikit-learn.org/stable/modules/generated/sklearn.model_selection.train_test_split.html), randomly dividing the data into a training set which is 80% of the overall data, reserving the remaining 20% for testing. Any division of the data will work here, as long as the training set is large enough for the machine learning model to learn from, but an 80-20 split is quite common.

Next we define our neural network using the sklearn function [`MLPClassifier`](https://scikit-learn.org/stable/modules/generated/sklearn.neural_network.MLPClassifier.html). "MLP" stands for multilayer perceptron and is another name for a neural network which has more than one layer, with a perceptron being another name for a neuron. Here we will start with a small neural network containing only one hidden layer of 10 neurons. We will train it for 50 epochs and use the relu activation function on just the hidden layer. After we define the neural network, stored in the variable `mlp`, we will fit it with the training data, meaning that the neural network will find the optimal values for its parameters by learning the patterns in the training data. We will then generate our predictions for the test data by giving the trained network only the X component of the test data, and having it predict what the outputs (the classes) should be. 

Finally, we compare the predicted classes of the test data, `y_pred`, to the known values, `y_test`. First we will compute an accuracy score, which determines the number of correct predictions as a ratio to the total number of predictions. An accuracy score of 1 means that all predictions were correct and an accuracy score of 0 means that none of them were correct. Multiplying the accuracy score by 100 makes the results a percentage of correct answers. We will use the sklearn function [`accuracy_score`](https://scikit-learn.org/stable/modules/generated/sklearn.metrics.accuracy_score.html) to determine the accuracy. 

We will also generalize the performance of our neural network using a [confusion matrix](https://en.wikipedia.org/wiki/Confusion_matrix), created using the sklearn function [`ConfusionMatrixDisplay`](https://scikit-learn.org/stable/modules/generated/sklearn.metrics.ConfusionMatrixDisplay.html). A confusion matrix helps us determine how many predictions were correct, how many predictions were not correct, and for the incorrect predictions, how were they wrong. In the confusion matrix below, the top left square is the number of people did not survive and were predicted to not survive. The bottom right corner are the people who were predicted to survive and did survive. The sum of these two squares are the correct predictions. The bottom left square represents people who did survive but were predicted to not survive and the top right square are people who were predicted to survive but did not survive. The sum of these two numbers are the number of incorrect predictions.

## Conclusion and Future Directions
In this activity, we built a neural network that learned to predict whether a passenger on the Titanic survived or not. Along the way, we went through the entire machine learning workflow that data scientists use in the real world. We started by **importing and cleaning** our data set, dropping columns we didn't need and converting words into numbers so our model could understand them. We then **visualized the data** with Seaborn to look for patterns, which helped us figure out which features (like passenger class, sex, and fare) were most useful for predicting survival.

Once our data was ready, we built our **first neural network** using just a single feature, age, and saw that it didn't perform very well. From there, we tried several different strategies to improve our accuracy:

* **Feature engineering and pre-processing:** Instead of one feature, we used several meaningful features and standardized them with `StandardScaler` so the network treated each one fairly.
* **Making the network deeper:** We rearranged our neurons into two hidden layers so the network could refine the data and find more complex patterns.
* **Making the network wider:** We tried adding more neurons to a layer to give the network more capacity to learn.
* **Changing the activation function and number of iterations:** We experimented with how the neurons fire and how long the network trains, while being careful to avoid underfitting (training too little) and overfitting (training so much the network just memorizes the data instead of learning general patterns).

The most important lesson here is that **building a good machine learning model is an iterative process.** There is rarely one perfect answer that works on the first try. Instead, data scientists make a change, measure the results using tools like the accuracy score and the confusion matrix, and then use what they learned to make the next change. Each of the choices we adjusted, like the number of layers, the number of neurons, the activation function, and the number of iterations, is a **hyperparameter**, and finding the best combination of them is called **hyperparameter tuning**.
